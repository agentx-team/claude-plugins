# wiki — 周报（每日记录→本周总结）与月报汇总算法

目标：
- 写「本周总结」时，**从同一周报文件的每日记录自动归纳**，不让用户重述；
- 写月报时，**从当月各周报自动归纳**。

---

## 数据模型

工作记录统一存于**周报文件** `$VAULT/journal/<YYYY>/W<ww>.md`（一周一个）。每个文件两部分：
- 顶部 `> [!abstract] 本周总结` callout（综述）；
- `## 📅 每日记录` 下若干 `### MM/DD (Day)` 块，每块含 `#### 🤝 客户互动`（表格 + `##### 会议纪要`）、`#### 📚 学习`、`#### 🔨 Build / Deliver`、`#### 💡 备注`。

解析方法：`grep -nE '^### [0-9]{2}/[0-9]{2}'` 定位每天块行号，再读区间内容。

---

## 本周总结（5A）

1. **确定周次**：默认本周；`TZ=Asia/Shanghai date +%V` 给 ISO 周号（周一起始），`date -d 'monday this week' +%Y-%m-%d` 等算边界。打开 `journal/<YYYY>/W<ww>.md`。
2. **读全部当天块**（同一文件内），归纳到「本周总结」callout：
   - **一句话主线**：本周最重要的 1-2 件事。
   - **关键交付**：汇总各天 `🔨 Build / Deliver`，去重合并同一项目多天进展，标状态（✅/🔄/❌）。
   - **客户互动 / 对外协作**：汇总各天 `🤝 客户互动` 表格与会议纪要，按客户归组。
   - **项目推进**：按项目（道通/CATL…）归类本周进展，合并跨天脉络。
   - **下周计划**：从未完成 Action Items（`- [ ]`）+ 用户补充提炼。
   - **风险与阻塞**：抽取明确卡点；无则省略。
3. **写回**：更新顶部「本周总结」callout，**不动每日记录明细**；可在 frontmatter 补 `summary:`。

> 这是**同文件内**的归纳——明细与总结都在 `W<ww>.md` 里，无需读外部日报文件。

## 月报汇总（5B）

1. **数据源**：当月覆盖的各周报 `journal/<YYYY>/W<ww>.md`。优先用各周「本周总结」做二次归纳（已去噪），需要细节再回看每日记录。
2. **归纳到月报各 section**（套 `_templates/月记模板.md`）：
   - **月度总结**：一句话概括本月。
   - **关键成果**：项目交付（表格）、技术成长（新技术/方案/文档输出）、对外协作。
   - **目标达成情况**：如能关联 SA 年度目标（如 vault 内 `projects/KPI/SA-GOAL.md`，存在时），对照填完成度。
   - **下月计划**：P0/P1/P2。
   - **风险与待解决**：跨月风险。
   - **月度复盘**：成就/挑战/教训/要改的习惯。
3. **写文件**：`$VAULT/journal/<YYYY>/M<MM>.md`（如 `M06.md`），frontmatter `type/monthly` + `month: <YYYY>-<MM>`。

---

## 归纳原则

- **去重合并**：同一项目跨多天进展合并成一条，体现脉络而非流水账。
- **抽象提升**：每日记录是细粒度动作，周/月总结提升到「交付/成果/影响」层面。
- **保留关键事实**：客户名、项目名、量化结果（如「30+ 企业」「TLI 候选」）要保留，不杜撰新数字。
- **空缺诚实**：某 section 无对应内容则留 `{待补充}` 或省略，不编。
- **可追溯**：本周总结来自本文件每日记录；月报来自各周报，形成层级。

## 不破坏既有内容

- 刷新本周总结只改顶部 callout，**保留每日记录明细**。
- 月报若已存在 → 默认合并补充新增内容，不覆盖；不确定时问用户。

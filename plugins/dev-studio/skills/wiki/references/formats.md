# wiki — 各类型文档格式参考

精确套用以下结构。结构源自 vault 的 `_templates/` 与既有工作周报 `journal/2026/W*.md`，先读那些实际文件确认最新风格，本文件是速查。

---

## 工作周报 — 单文件结构（每日记录 + 本周总结）

工作记录的唯一载体：`$VAULT/journal/<YYYY>/W<ww>.md`（一周一个文件）。整体骨架：

```markdown
---
tags:
  - type/weekly
  - project/KPI
week: "<YYYY>-W<ww>"
created: "<周一日期>"
modified: "<更新日期>"
---

# <YYYYMM> ... 实为 -> # <YYYY>-W<ww> 周报

**周期**：<周一> ~ <周日>　|　**记录天数**：N 天

> [!abstract] 本周总结
> {从下方每日记录归纳：关键交付 / 客户互动 / 项目推进 / 一句话主线 / 下周计划 / 风险}

---

## 📅 每日记录

### MM/DD (Day) ☀️

#### 🤝 客户互动

| 客户 | Topic | 形式 | OPP ID | Next Action |
|------|-------|------|--------|-------------|
| {客户名} | {讨论主题} | {线上/线下/电话/现场} | {OPP ID 或 N/A} | {后续行动} |

##### 会议纪要 — {客户名} {主题}

- 参会人：
- 关键讨论点：
  1.
- 客户反馈：
- Action Items：
  - [ ]

#### 📚 学习

-

#### 🔨 Build / Deliver

1. **{项目/产出名}**
   - {要点}

#### 💡 备注

-

---

### MM/DD (Day) ☀️
... 下一天，同结构 ...

---
```

**要点：**
- 一周一个文件 `W<ww>.md`；当天记录写到「📅 每日记录」下，**按日期逆序（最新一天在最上面）**——新的一天插在「📅 每日记录」标题正下方，各天用 `---` 分隔。
- 标题层级：每天 `### MM/DD (Day)` → 板块 `#### 🤝/📚/🔨/💡` → 会议纪要 `##### 会议纪要 — …`。
- 无内容的 section 直接省略；`Day` 用英文星期缩写（Mon…Sun）。
- 多个客户互动 → 表格多行；多个会议 → 多个 `##### 会议纪要` 子节。
- 「本周总结」由 §5A 从每日记录归纳，**与明细同住一文件**；写当天记录后顺手刷新。
- 新建文件时，`created` = 该周周一日期，frontmatter 用 `type/weekly` + `project/KPI`。

## 个人日记 — 文件

文件：`$VAULT/journal/<YYYY>/<YYYYMMDD>.md`（套 `_templates/日记模板.md`）

关键 section：`✅ 今日任务`、`💻 技术工作`（完成的工作 / 问题与解决 / 关键代码）、`📖 学习与收获`、`📝 临时笔记`、`🔄 复盘`。frontmatter `type/journal`。

---

## 通用笔记

无法归类时落 `$VAULT/_raw/<kebab-slug>.md`，最简 frontmatter（`type/note` + 日期），正文按内容自然组织。日后可由蒸馏（本技能 §6）或 `framework/wiki-ingest/SKILL.md` 提升。
